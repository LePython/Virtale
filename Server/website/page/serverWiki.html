This is the Documentation for the VirtaleVR Server

Some general information

The Server currently runs on port 8080 and on the localhost. A configuration file will be added in the near future.

!!Make sure you installed python3 and pip3 otherwise nothing will work!!

Installation

Linux

  • download the server, the necessary scripts and the model
  • put the zip file in the same folder and unzip them there
  • now delete the zip files
  • go to the scripts subfolder and run the install_linux.sh if the installation fails run su_install_linux.sh if both fail try to somehow install all dependencies in the requirements.txt manually
  • at last you can run the executable (main.out)
  • If you alreday have some audio files and want to have them analyzed put them in the audio_files folder and run CategorizeSong.py in the scripts folder.

Windows

Currently windows is not tested and officially supported. If you want to you can try to build and run the code from here. All the necessary files for the Server are here

How to update if you have already installed a version?

Linux

  • First stop the running server
  • Download the ServerUpdate and put the files in the same folder as the main.out
  • Download the UpdateScript and put it where the Update.zip is
  • Run the update_linux.sh
  • Now you can start the Server again :smile:

What can you analyze at the moment?

Currently it is simply possible to analyze new songs from Youtube and fetch Songs that were previously analyzed.

How to analyze songs from Youtube?

To analyze a song form Youtube you need to do a simple HTTP GET request to the analyzeYTHandler subdomain. The body should be JSON and should have the following form: { "url":"some_youtube_link", "format":"file_format" }

The Server will download the video and save it in the given file_format. After that it will be analyzed and the Server will return the url, filename and featuregroup of the Link.

How to get an audio file from the server?

Requesting a audio file is done by simply making a GET request with a JSON body to the subdomain getAudio It contains the just one Tag, the filename:"name" The server will serve the file as a response.

Getting a list that contains all currently analyzed features?

Simply to a HTTP GET request to the getSongList subdomain :grin:.
